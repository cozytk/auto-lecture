# Git 기본 명령어

```bash {all|1-3|5-8|10-12}
# 1. 저장소 초기화
mkdir my-project
cd my-project
git init

# 2. 파일 추가 및 스테이징
echo '# My Project' > README.md
git add README.md
git status

# 3. 커밋
git commit -m '프로젝트 초기화'
git log --oneline
```

<v-clicks>

- `git init`: 현재 디렉토리를 Git 저장소로
- `git add`: Working → Staging
- `git commit`: Staging → Repository

</v-clicks>

[Q&A 대비]
Q: git add . 처럼 점을 찍으면 모든 파일이 한 번에 올라가는데, 그냥 항상 이걸 쓰면 안 되나요?
A: `git add .`은 현재 디렉토리의 모든 변경 파일을 스테이징합니다. 혼자 연습할 때는 편하지만, 실무에서는 위험할 수 있습니다. 예를 들어 비밀번호가 들어있는 .env 파일이나 빌드 결과물 같은 걸 실수로 커밋할 수 있습니다. 파일명을 직접 지정하는 습관을 들이시는 게 좋고, 최소한 `git status`로 어떤 파일이 올라가는지 확인한 뒤에 add를 하시기 바랍니다.

Q: commit 메시지 없이 커밋할 수는 없나요? 매번 메시지 쓰는 게 귀찮습니다.
A: `-m` 플래그를 빼고 `git commit`만 치면 에디터가 열리면서 메시지를 입력하라고 합니다. 메시지를 아예 비울 수는 있지만, 강력히 권장하지 않습니다. 나중에 커밋 이력을 봤을 때 "이 커밋이 뭘 한 건지" 전혀 알 수 없게 됩니다. 특히 팀으로 작업하면 동료가 여러분의 커밋 메시지를 보고 변경 내용을 파악해야 하기 때문에, 간결하더라도 반드시 적는 습관을 가지시기 바랍니다.
