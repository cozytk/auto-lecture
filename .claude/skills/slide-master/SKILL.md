# 슬라이드 마스터

강의 가이드를 slidev 프레젠테이션으로 변환한다. 가이드의 요약이 아니라 발표의 시각적 보조 도구를 만든다.

> **Slidev 문법 참조**: 구문, 레이아웃, 애니메이션, 코드 하이라이트 등 Slidev 기술 상세는 `slidev` 스킬(`/slidev`)과 그 references 디렉토리를 참조한다. 이 스킬은 교수법적 설계 판단에 집중한다.

## 입력 및 산출물

- 입력: `lectures/{topic}/guide/` 디렉토리 (없으면 에러 출력 후 중단)
- 산출물: `lectures/{topic}/slides/slides.md`, `lectures/{topic}/slides/assets/`

## 작업 순서

1. 가이드 전체 파일 읽기 → 구조 분석
2. 슬라이드 구성 설계 (슬라이드 목록 + 핵심 메시지)
3. WebSearch로 이미지 검색 → assets 저장
4. slides.md 작성 → 자가 검증 → 완료 보고

## 핵심 원칙

### 슬라이드 구조

1. **한 슬라이드 = 하나의 메시지.** 두 가지 이상의 개념을 한 장에 넣지 않는다. 슬라이드는 많을수록 좋다 — 압축하지 말고 분할한다.

2. **넘침 금지.** 캔버스는 980×552px 고정이다. 넘칠 것 같으면 분할한다. 불릿 최대 5개, 코드 블록 최대 15줄, 본문 300자. 약간 넘칠 때는 `zoom: 0.9`(0.8 미만 금지). 하단 잘림에 주의한다.

3. **제목은 한 줄.** 한글 20자/영문 40자 초과 금지. 개념/섹션 제목은 `1. {제목}` 형식.

4. **전환·휴식 슬라이드**(점심시간, 쉬는 시간 등)는 간결하게. 불릿이나 배경 박스 없이 제목 + 시간 + 간단한 안내만.

### 시각 설계

5. **v-click으로 점진적 공개.** 불릿, 카드, 코드 설명은 한 번에 보여주지 않는다. `<div>` 내부에서 `<v-clicks>`를 쓸 때는 전후에 빈 줄 필요.

6. **가이드의 "왜"를 슬라이드의 "무엇"으로.** 자세한 설명은 발표자 노트에, 슬라이드엔 키워드와 코드만.

7. **텍스트 색상이 기본, 배경 박스는 예외.** 강조에는 `text-green-600`(긍정), `text-red-500`(부정), `text-blue-600`(정보) + 아이콘(✅ ❌ ⚠️)을 사용한다. 배경 박스(`bg-{color}-50 rounded-lg`)는 슬라이드의 핵심 결론 1개에만 허용하며, **코드 블록이 있는 슬라이드에서는 사용 금지**. "흔한 실수", "팁", "비교" 같은 교육 내용은 색상 텍스트로 충분하다.

8. **시각적 위계로 정보를 구분한다.** 텍스트 크기(`text-2xl` vs `text-sm`), 굵기(`font-bold`), 색상, 들여쓰기로 상위/하위를 표현한다. 핵심 메시지는 `text-xl font-bold` 이상.

9. **시각 자료를 적극 삽입한다.** 텍스트 전용 슬라이드가 3장 연속되면 Mermaid, 이미지, 플레이스홀더를 삽입한다. Mermaid는 `style` 지시어로 노드에 색상을 지정하고, 노드 10개 이하, `scale` 0.5~0.8로 넘침을 방지한다.

10. **비교 슬라이드는 좌우 색상을 다르게.** 대비되는 개념은 동일 크기·위치로 배치하되 다른 색상 계열로 구분한다. `two-cols` 또는 `two-cols-header` 레이아웃에 충분한 column-gap을 확보한다.

### Slidev 렌더링 제약 (기술적 필수)

11. **HTML 태그 내부에서 `**bold**` 금지.** `<div>`, `<v-click>`, `<v-clicks>` 등 HTML/Vue 태그 안에서는 `<strong>` 또는 UnoCSS(`font-bold`)를 사용한다. Slidev의 마크다운 렌더러는 HTML 블록 내부의 `**...**`를 변환하지 않는다.

12. **Markdown 번호 목록(`1.`, `2.`)을 슬라이드 본문에 쓰지 않는다.** Slidev에서 부제목처럼 렌더링된다. `<v-clicks>` + 불릿(`-`)이나 원기호(①②③)를 사용하되, 불릿과 원기호를 혼용하지 않는다.

13. **발표자 노트의 `[click]` 수 = 슬라이드의 v-click 단계 수.** 반드시 영어 `[click]`을 사용한다(`[클릭]` 금지).

### 코드 블록

14. **라인 하이라이트 범위는 논리적 코드 단위와 일치시킨다.** 각 `|` 단계가 함수, 객체, 블록의 중간에서 끊기면 안 된다. 작성 후 줄 번호를 세어 각 범위의 마지막 줄이 닫는 괄호이거나 독립 문장인지 검증한다.

15. **15줄 초과 코드 블록은 분할하거나 `{maxHeight:'380px'}`를 추가한다.** 라인 하이라이트와 결합 시: `{1-10|12-20}{maxHeight:'380px'}` (공백 없이).

### 퀴즈

16. **퀴즈 정답을 시각적으로 노출하지 않는다.** 모든 선택지를 동일 배경색으로 초기 표시하고, v-click으로 정답을 공개한다. 구현 상세는 Slidev `v-click-hide` 문법을 Context7로 확인하라.

## 폰트 지침 (Freesentation)

`lectures/{topic}/slides/style.css`에 Freesentation 폰트를 적용한다. 없으면 생성.

- `assets/fonts/`에 ttf 파일이 없으면 시스템 폰트(`~/Library/Fonts/Freesentation*.ttf`)에서 복사
- 9개 웨이트(100~900) 전체를 `@font-face`로 등록

```css
@font-face {
  font-family: 'Freesentation';
  src: url('./assets/fonts/Freesentation-{N}{Name}.ttf') format('truetype');
  font-weight: {weight};
  font-style: normal;
}
```

### 요소별 웨이트

| 요소 | Weight |
|------|--------|
| `h1` | 900 (Black) |
| `h2` | 700 (Bold) |
| `h3` | 600 (SemiBold) |
| `h4`~`h6` | 500 (Medium) |
| 본문 | 400 (Regular) |
| `strong`, `b` | 800 (ExtraBold) |
| 작은 텍스트 (`.text-sm`, `.text-xs`) | 300 (Light) |
| `blockquote`, cover/section 부제 | 200 (ExtraLight) |

## 스타일링 규칙

UnoCSS(Tailwind 호환) 내장. 전체 문법은 slidev 스킬 references 참조.

### 필수 패턴
- 이미지: `max-h-[380px]`(전체) / `max-h-[320px]`(two-cols 내)
- 넘침 방지: 본문 컨테이너에 `overflow-hidden`
- 하단 고정 정보: `absolute bottom-6 left-14 text-xs opacity-50`
- 텍스트 색상 강조(기본): `text-{color}-600` + 아이콘(✅ ❌ ⚠️)
- 배경 박스(예외적): `bg-{color}-50 dark:bg-{color}-900/30 rounded-lg p-4` — 슬라이드당 최대 1개, 코드 블록 공존 금지 (규칙 7)

## 애니메이션 지침

구문 상세는 slidev 스킬 references 참조.

| 콘텐츠 | 애니메이션 | 비고 |
|---------|-----------|------|
| 불릿 리스트 | `v-clicks` 순차 공개 | 필수 |
| 코드 블록 | 라인 하이라이트 (`{1\|3-5}`) | 필수 |
| 코드 진화 과정 | Magic Move | 단순 코드는 라인 하이라이트로 충분 |
| 핵심 수치·카드 | `v-motion` | 적극 활용 |
| 키워드 강조 | `v-mark` | 적극 활용 |

- 전환: cover/section은 `fade`, 일반은 `slide-left`
- 모션 딜레이 최대 400ms, 한 슬라이드에 v-click + v-motion + v-mark 동시 사용 금지

## 발표자 노트 (완전한 수업 스크립트)

모든 슬라이드에 HTML 주석으로 **그대로 읽어도 수업을 진행할 수 있는 완전한 스크립트**를 작성한다.
강사가 해당 주제를 잘 모르더라도 스크립트를 읽는 것만으로 자연스럽게 수업을 진행할 수 있어야 한다.

```markdown
<!--
[스크립트]
{도입: 이전 슬라이드/개념과의 연결. "방금 X를 배웠습니다. 그런데..." 또는 "앞에서 Y를 봤는데..."}

{본문: 이 슬라이드에서 강사가 말할 전체 내용. 완성된 문장으로, 구어체로 작성.
v-click이 있으면 [click] 표시와 함께 각 단계별 설명을 구분한다.

- 가이드의 비유/아날로지를 활용하여 풀어서 설명한다
- 수강생 배경(audience)에 맞는 비교를 포함한다
- "여기서 혼란스러울 수 있는 부분이 있습니다" 식으로 오해를 선제적으로 짚는다}

[click] {각 v-click 단계마다 충분한 설명. 코드라면 줄 단위로 해설}

[Q&A 대비]
Q: {예상 질문 1 — 자주 묻는 질문}
A: {3~5문장 답변. 이유와 맥락 포함}

Q: {예상 질문 2 — 수강생 배경에서 나올 질문}
A: {익숙한 개념과 비교하며 답변}

Q: {논쟁적 포인트가 있으면}
A: {양쪽 시각을 공정하게 소개}

전환: {다음 슬라이드로 넘어가는 연결 멘트. 다음 개념을 예고하는 문장}
시간: X분
-->
```

### 스크립트 작성 원칙

- 1분당 약 200자 기준으로 할당 시간에 맞게 작성
- 슬라이드에 없는 배경, 이유, 실무 사례를 풍부하게 보충
- Q&A 대비 최소 2개 (자주 묻는 질문 1개 + 수강생 배경 질문 1개)
- **가이드의 6단계 설명**(비유, 오해, 비교 등)을 발표용 구어체로 변환한다
- 가이드에 비유/아날로지가 있으면 스크립트에서 활용한다
- 코드 슬라이드에서는 **줄 단위 해설**을 포함한다 ("첫 번째 줄을 보면...")
- "전환" 멘트는 단순 "다음으로 넘어가겠습니다"가 아니라, **다음 개념이 왜 필요한지**를 1문장으로 설명한다

## 이미지·다이어그램 규칙

시각 자료는 선택이 아니라 필수다.

- **아키텍처·흐름**: Mermaid 다이어그램 (Context7로 문법 확인)
- **제품·도구**: WebSearch → assets 저장, 실패 시 `<!-- IMAGE: {설명}. 검색: "{쿼리}" -->`
- **텍스트 3장 연속 금지**
- 이미지에는 반드시 `max-h` 클래스 지정

## 자가 검증

작성 완료 후 모든 슬라이드를 순회하며 확인한다:

1. 오버플로: 넘침·하단 잘림 없는가? 이미지에 `max-h` 있는가? (규칙 2)
2. 발표자 노트: 완전한 스크립트가 있고, `[click]` 수가 v-click 단계와 일치하는가? (규칙 13)
3. 애니메이션: v-click 또는 라인 하이라이트가 사용되었는가? (규칙 5)
4. HTML 내 볼드: `**...**`가 HTML/Vue 태그 안에서 쓰이지 않았는가? (규칙 11)
5. 배경 박스: 슬라이드당 1개 이하이고, 코드 블록과 공존하지 않는가? (규칙 7)
6. 코드 하이라이트: 범위가 논리적 코드 단위의 경계와 일치하는가? (규칙 14)
7. 퀴즈: 정답이 초기 상태에서 색상으로 노출되지 않는가? (규칙 16)
8. 폰트: `style.css`에 Freesentation 폰트 설정이 있는가?
9. 시각 자료: 텍스트 전용 슬라이드가 3장 연속되지 않는가? (규칙 9)

### 스크립트-슬라이드 싱크 검증

- 스크립트에서 언급한 시각 요소는 실제 슬라이드에 존재해야 한다.
- 스크립트 분량은 `시간: X분` 기준 1분당 약 200자.

### Playwright 검증

가능하면 Slidev를 렌더링한 뒤 Playwright로 시각 검증을 수행한다. 잘림, 오버플로, 겹침, 누락된 이미지를 우선 확인한다.

위반 항목은 수정 후 완료 보고:

```
슬라이드 생성 완료: lectures/{topic}/slides/slides.md
- 총 슬라이드 수: N장
- 이미지: N개 (placeholder: N개)
- 애니메이션: v-click N / v-motion N / magic-move N
- 총 예상 발표 시간: N분
```
