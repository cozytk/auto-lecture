## README 중심 실습

분석, 설계, 의사결정, 토론 실습에 사용한다.

```markdown
# {실습 제목}

## 실습 목적
{해당 실습에서 확인하는 핵심 개념}

## 산출물
- 워크시트 / 의사결정표 / 설계 초안 / 비교표 / 회고 메모 등

## I DO: 시연 관찰 (약 N분)
- 강사가 문제를 어떻게 구조화하는지 시연
- 관찰 포인트 제시

## WE DO: 함께 실습 (약 N분)
- 강사와 함께 체크리스트, 표, 템플릿의 일부를 완성
- 각 단계마다 왜 이런 판단을 했는지 토론 질문 포함

## YOU DO: 독립 과제 (약 N분)
- 학생이 남은 항목을 스스로 완성
- 제출 기준과 자기 점검 기준 제시

## 작성 템플릿
{표/체크리스트/워크시트 템플릿}

## 검증 방법
- 산출물이 필수 항목을 모두 포함하는가
- 판단 근거가 구체적인가
- 다른 세션 실습과 자연스럽게 연결되는가

## 트러블슈팅
| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| ...  | ...  | ...      |
```

## Justfile 패턴

코드 실습일 때만 작성한다. README 중심 실습에는 Justfile을 만들지 않는다.

```justfile
# {실습 제목} 자동화 스크립트

# 환경 준비: 의존성 설치 및 초기 설정
setup:
    {환경 준비 명령어}

# 실행: 실습 메인 로직 실행
run:
    {실행 명령어}

# 검증: 실습 결과 검증
test:
    {검증 명령어}

# 정리: 실습 환경 초기화
clean:
    {정리 명령어}
```

## src/ 디렉토리 구조

코드 실습일 때만 생성한다.

```
src/
  i-do/           # I DO: 강사 시연용 완성 코드
  we-do/          # WE DO: 함께 실습용 스캐폴드 (TODO 포함)
  you-do/         # YOU DO: 독립 과제 템플릿 (TODO 포함)
```

- **i-do/**: 완성된 동작 코드. 강사가 실행하며 시연.
- **we-do/**: 핵심 로직이 `TODO:` 주석으로 비어 있는 스캐폴드. 강사와 함께 채운다.
- **you-do/**: 새로운 요구사항이 주어진 과제 템플릿. `TODO:` 주석으로 작성할 부분을 표시.
- 모든 코드는 실제 동작 가능한 상태로 작성한다.

## solution/ 디렉토리

YOU DO 과제의 정답 코드를 반드시 제공한다.

```
solution/
  {you-do 과제 파일과 동일한 구조}
```

- YOU DO 템플릿의 `TODO:` 부분이 모두 채워진 완성 코드
- 정답 코드에는 핵심 로직에 대한 간단한 주석을 포함하여 학습 효과를 높인다
- README 중심 실습에서는 코드 정답 대신 `artifacts/` 하위 예시 문서 또는 README 내 모범 답안으로 대체 가능
