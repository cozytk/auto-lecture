# 실습 품질 체크리스트

실습 제작 완료 전 아래 항목을 확인한다. 모든 항목을 만족해야 완료 보고할 수 있다.

## 구조 검증

- [ ] 모든 실습에 README.md가 존재하는가
- [ ] 모든 README.md에 I DO → WE DO → YOU DO 3단계가 있는가
- [ ] 각 단계에 예상 소요 시간이 명시되어 있는가
- [ ] 코드 실습의 경우 Justfile이 존재하는가
- [ ] 코드 실습의 src/에 i-do/, we-do/, you-do/ 하위 디렉토리가 있는가
- [ ] YOU DO에 대한 정답(solution/ 또는 artifacts/)이 존재하는가

## 내용 검증

- [ ] 가이드의 실습 포인트와 1:1 대응되는가 (빠진 실습이 없는가)
- [ ] 실습 시간 합산이 해당 day 전체의 약 70%인가
- [ ] 사전 준비에 필요한 도구/버전이 명확한가
- [ ] WE DO의 TODO 주석이 구체적이고 단계가 명확한가
- [ ] YOU DO 과제가 WE DO의 단순 반복이 아닌 변형/확장인가
- [ ] 힌트가 충분하되, 정답을 직접 알려주지 않는가

## 코드 품질 (코드 실습인 경우)

- [ ] i-do/ 코드가 에러 없이 실행되는가
- [ ] we-do/ 스캐폴드가 TODO를 채우면 동작하는가
- [ ] solution/ 코드가 에러 없이 실행되는가
- [ ] `just setup && just run && just test`가 통과하는가
- [ ] 주석이 "왜(Why)"를 설명하는가 (코드 번역 주석 없음)

## 기술 정확성

- [ ] 사용한 라이브러리 버전이 공식 문서와 일치하는가
- [ ] API, CLI 명령어, 설정값이 현재 버전에서 유효한가
- [ ] 트러블슈팅 섹션이 현실적인 문제를 다루는가

## 학습 설계

- [ ] 난이도가 점진적으로 증가하는가 (기초 → 중급 → 심화)
- [ ] 코드 실습과 README 중심 실습이 적절히 섞여 있는가
- [ ] 실습 간 선행 조건이 명확한가 (순서 의존성)
