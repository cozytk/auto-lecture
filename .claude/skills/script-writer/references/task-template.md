# 발표자 대본 작성 태스크 템플릿

이 파일은 병렬 워커(Agent subagent 또는 Codex)에게 전달할 태스크 파일의 템플릿이다.
오케스트레이터는 아래 템플릿의 `{placeholder}`를 실제 값으로 채워 태스크 파일을 생성한다.

> 각 태스크 파일은 **자기 완결적(self-contained)**이어야 한다 — 워커가 다른 파일을 읽을 필요 없이 태스크 파일만으로 대본을 작성할 수 있어야 한다.

---

## 템플릿

~~~markdown
# 발표자 대본 작성 태스크

## 기본 정보
- 주제: {topic}
- 강의 제목: {lecture title}
- 담당 세션: Session {n} — {session title} ({예상 시간}분)
- 수강 대상: {audience}
- 전체 세션 수: {total sessions} (이 태스크는 {n}/{total} 번째)

## 출력 지침

**출력 파일**: `{절대경로}/.omc/script-writer/{topic}/session-{n}-output.md`

### 출력 형식
- 아래 "작성 대상 슬라이드"의 **원문 Markdown을 그대로 보존**한다
- 각 슬라이드에 `<!-- [스크립트] ... -->` HTML 주석 블록을 추가하거나, 기존 placeholder를 교체한다
- 슬라이드 구분자 `---`와 YAML frontmatter도 그대로 유지한다
- **출력 파일 외에 다른 파일은 절대 수정하지 않는다**

## 대본 작성 규칙

{여기에 references/writing-rules.md의 전문을 복사한다}
{요약하거나 축약하지 않는다 — 워커가 규칙을 완전히 이해해야 고품질 대본이 나온다}

## 세션 연결 정보

### 이전 세션 마지막 내용
{이전 세션의 마지막 슬라이드 제목 + 핵심 내용 1~2문장 요약}
{첫 세션이면: "없음 — 강의 시작. 인사와 수업 소개로 시작한다."}

### 다음 세션 첫 내용
{다음 세션의 첫 슬라이드 제목 + 핵심 내용 1~2문장 요약}
{마지막 세션이면: "없음 — 강의 종료. 정리와 Q&A로 마무리한다."}

## 참고 가이드 (해당 세션)

```
{해당 세션에 관련된 가이드 원문을 여기에 붙여넣기}
```

## 작성 대상 슬라이드

```
{해당 세션의 슬라이드 원문을 --- 구분자 포함하여 그대로 붙여넣기}
{기존 <!-- --> 주석이 있으면 그대로 포함 — 워커가 교체할 대상}
```
~~~
