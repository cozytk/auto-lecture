## v2 실습 리뉴얼 설계 결정 (2026-03-18)

### 리뉴얼 배경

v1 실습의 문제점:
- TODO 앱이 3개 실습에서 반복 사용되어 단조로움
- 관찰/비교 위주로 "직접 만드는" 경험 부족
- Lab 03이 15분으로 너무 짧음
- 캡스톤이 설계만 하고 실제 빌드 없음

참고한 외부 커리큘럼:
- MIT Missing Semester 2026 Agentic Coding Module
- GitHub Copilot 4-Week Bootcamp (Pwd9000)
- Stanford Vibe Coding for Developers
- Tactical Agentic Coding (8-lesson course)

### 각 실습별 프로젝트 (모두 독립적, TODO 앱 미사용)

- **Lab 00**: CLI 마크다운 메모장 `memonote` (5개 파일, Python 표준 라이브러리)
  - 의도적으로 문서화가 부족하여 에이전트가 코드를 읽어야 파악 가능
  - 24개 unittest 통과
- **Lab 01**: 북마크 관리 REST API (http.server 기반, 외부 패키지 없음)
  - 3라운드 비교: 규칙 없음(3개 테스트) → 기본 규칙(13개) → 풍부한 컨텍스트(15개)
  - 동일 요청("category 필드 추가")으로 컨텍스트 효과를 정량 비교
- **Lab 02**: 데이터 분석 레거시 스크립트 (의도적으로 지저분한 ~280줄 단일 파일)
  - 거대 함수, camelCase/snake_case 혼용, 매직 넘버, 중복 코드, 에러 핸들링 없음
  - 솔루션: 5개 모듈로 분리, 17개 테스트, 에러 처리 완비
  - empty.csv에서 레거시는 ZeroDivisionError, 솔루션은 우아한 에러 처리
- **Lab 03**: CLI 단어장 앱 (영어 학습 도구)
  - 의도적 버그 3개: 타입 에러(stats), 대소문자 로직(quiz), 데이터 무결성(vocab)
  - 10개 테스트 중 정확히 3개 실패 → 솔루션에서 10/10 통과
  - 자동 수정 루프 스크립트 템플릿 제공 (학생이 완성)
- **Lab 04**: 바이브 코딩 캡스톤 (6개 시나리오 선택)
  - 코드를 한 줄도 직접 쓰지 않고 프롬프트만으로 앱 완성
  - 예시 프로젝트(포모도로): 15개 테스트, `--duration 5`로 빠른 데모 가능

### 핵심 개선 포인트

1. **프로젝트 다양성**: 5개 실습이 모두 다른 프로젝트 (메모장, API, 데이터분석, 단어장, 자유)
2. **능동적 참여**: 관찰→비교에서 탐험→구축→자동화→빌드로 전환
3. **정량적 비교**: Lab 01에서 토큰/도구호출/품질을 숫자로 비교
4. **에이전트 한계 체험**: Lab 02에서 동어반복 테스트 발견, Lab 03에서 자동 수정 한계
5. **성취감 있는 마무리**: Lab 04에서 실제 동작하는 앱을 만들고 시연
