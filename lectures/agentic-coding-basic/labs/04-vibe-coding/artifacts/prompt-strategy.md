# 바이브 코딩 프롬프트 전략 가이드

바이브 코딩에서 성패를 가르는 것은 코딩 실력이 아니라 **프롬프트 전략**이다.
이 가이드를 따르면 40분 안에 동작하는 앱을 완성할 수 있다.

---

## 4단계 전략

### Step 1: 프로젝트 구조 프롬프트

**목표**: 파일과 디렉토리 뼈대를 먼저 만든다.

```
[시나리오 설명을 붙여넣기]

위 요구사항을 구현할 Python 프로젝트를 만들어줘.
지금은 빈 파일만 만들어도 돼. 각 파일에 모듈 역할을 주석으로만 써줘.
파일 구조는 이렇게 해줘:
- pomodoro.py (메인 진입점)
- timer.py (타이머 로직)
- storage.py (세션 저장)
- stats.py (통계)
- test_pomodoro.py (테스트)
```

**왜 이렇게 하나?**
전체 구조를 먼저 합의하면, 이후 각 파일을 채울 때 에이전트가 맥락을 이해하고 일관성 있는 코드를 작성한다.

---

### Step 2: 핵심 기능 프롬프트 (하나씩, 구체적으로)

**목표**: 기능을 한 번에 하나씩 구현한다.

**좋은 예시**:
```
storage.py를 구현해줘.
- 세션 데이터를 sessions.json에 저장
- save_session(start_time, end_time, session_type, completed) 함수
- load_sessions() → 전체 세션 목록 반환
- sessions.json이 없으면 빈 리스트로 초기화
- 표준 라이브러리만 사용 (json, datetime)
```

**나쁜 예시** (안티패턴):
```
포모도로 타이머 앱 전체를 만들어줘. 타이머, 저장, 통계 다 포함해서.
```

**왜 나쁜가?** 너무 큰 요청은 에이전트가 구조를 임의로 결정하고, 요구사항 일부를 빠뜨리거나, 테스트하기 어려운 코드를 만든다.

---

### Step 3: 테스트 프롬프트

**목표**: 구현이 끝난 모듈에 바로 테스트를 추가한다.

```
test_pomodoro.py에 storage.py 테스트를 추가해줘.
- 세션 저장 후 불러오기가 정확한지 확인
- 여러 세션 저장 후 개수가 맞는지 확인
- 임시 파일을 사용해서 실제 sessions.json을 오염시키지 않도록
- unittest 사용
```

**팁**: 테스트는 구현 직후에 요청해야 한다. 나중에 한꺼번에 하면 에이전트가 각 모듈의 상세 구현을 잊는다.

---

### Step 4: 개선 프롬프트

**목표**: 동작하는 기본 버전을 먼저 완성한 후 개선한다.

```
pomodoro.py의 타이머 출력을 개선해줘.
지금은 매 초 새 줄로 출력되는데, \r을 사용해서 같은 줄에서 업데이트되도록 수정해줘.
형식: [=========>         ] 18:32 남음
```

```
에러 처리를 추가해줘.
sessions.json 파일이 손상됐을 때 (JSON 파싱 오류) 빈 리스트로 초기화하고
사용자에게 경고 메시지를 출력하도록 storage.py를 수정해줘.
```

---

## 에러 처리 프롬프트

에러가 발생했을 때 에이전트에게 전달하는 방법:

```
아래 에러가 발생했어. 분석하고 수정해줘.

[에러 메시지 전체 붙여넣기]

실행 명령어: python pomodoro.py start
```

**중요**: 에러 메시지를 요약하지 말고 전체를 복사해서 붙여넣는다.
스택 트레이스에 중요한 정보가 있다.

## 효과적인 프롬프트 패턴

### 제약 조건 명시
```
표준 라이브러리만 사용해줘. (json, datetime, os, sys, time, unittest만 허용)
```

### 인터페이스 먼저 합의
```
stats.py의 함수 시그니처를 먼저 정의해줘. 구현은 아직 안 해도 돼.
- get_today_sessions() → List[Session]
- get_streak() → int
- get_total_focus_minutes(date: str) → int
```

### 단계별 검증 요청
```
storage.py 구현 후, 간단한 수동 테스트 코드도 같이 작성해줘.
__main__ 블록에서 직접 실행해볼 수 있도록.
```

### 리팩토링 요청
```
pomodoro.py가 너무 길어졌어. 타이머 출력 관련 코드를 display.py로 분리해줘.
기존 기능은 그대로 유지해야 해.
```

## 시간 관리 팁

40분 안에 완성하려면 기능 우선순위를 정해야 한다.

**추천 시간 배분 (40분 기준)**:

| 시간 | 활동 |
|------|------|
| 0~5분 | AGENTS.md 작성 + 프로젝트 구조 요청 |
| 5~25분 | 핵심 기능 3개 구현 (기능당 ~7분) |
| 25~35분 | 테스트 작성 + 실행 확인 |
| 35~40분 | 발표 준비 (시연 흐름 정리) |

**20분이 지났는데 1개 기능도 완성 못 했다면**: 범위를 줄여라. 완성된 1개가 미완성 5개보다 낫다.
