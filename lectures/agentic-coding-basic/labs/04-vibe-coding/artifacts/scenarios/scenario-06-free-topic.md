# 시나리오 6: 자유 주제

**난이도**: 자유
**예상 완성 시간**: 팀이 설정한 범위에 따라 다름
**언어**: 자유 (팀 합의)

---

## 개요

팀이 원하는 프로젝트를 자유롭게 선택해 바이브 코딩으로 완성한다.
단, 아래 최소 요구사항을 반드시 충족해야 한다.

---

## 최소 요구사항 (필수)

### 1. 기능 요건
- **최소 3개의 독립적인 기능** 구현
- 각 기능은 CLI 명령어 또는 뚜렷한 사용자 액션으로 구분됨
- 실제 데이터를 저장하거나 처리하는 기능 포함

### 2. 테스트 요건
- 핵심 기능에 대한 **자동화 테스트 최소 5개**
- `pytest` 또는 언어 기본 테스트 프레임워크 사용
- 테스트 실행 방법이 README에 명시되어야 함

### 3. AGENTS.md 요건
- 프로젝트 루트에 `AGENTS.md` 파일 존재
- 최소 포함 항목:
  - 프로젝트 개요 (1~3줄)
  - 사용 언어 및 버전
  - 테스트 실행 명령어
  - 코딩 규칙 2개 이상

---

## 아이디어 예시

아이디어가 없다면 아래에서 영감을 얻어라:

### 간단한 유틸리티
- **단어 빈도 분석기**: 텍스트 파일을 받아 단어 빈도 Top N 출력
- **파일 이름 일괄 변경기**: 패턴 기반 파일 이름 변경 (날짜 추가, 공백 → 언더스코어 등)
- **TODO 파일 파서**: 코드베이스에서 `// TODO` 주석을 찾아 목록화
- **간단한 암호화 도구**: Caesar cipher 또는 간단한 치환 암호

### 데이터 처리
- **CSV 분석기**: CSV 파일 로드, 컬럼 통계(평균/최대/최소), 필터링
- **로그 분석기**: 로그 파일 파싱, 에러 패턴 추출, 시간대별 분포
- **JSON 포매터/검증기**: JSON 파일 예쁘게 출력, 스키마 검증

### 게임 / 재미
- **터미널 퀴즈 게임**: 문제 파일 기반 OX 또는 객관식 퀴즈
- **타이핑 연습 앱**: 랜덤 문장 표시, 속도/정확도 측정
- **터미널 틱택토**: 2인 대전 또는 AI 상대

### 생산성
- **간단한 일기 앱**: 날짜별 텍스트 저장, 키워드 검색
- **북마크 매니저**: URL + 태그 + 메모 저장, CLI로 검색
- **회의록 템플릿 생성기**: 참석자/안건 입력 → Markdown 템플릿 생성

## 범위 설정 가이드

바이브 코딩 40분 안에 완성할 수 있는 범위로 제한한다.

| 팀 수준 | 권장 범위 |
|---------|---------|
| 입문 | 단일 파일, 기능 3개, 데이터는 파일에 저장 |
| 중급 | 모듈 분리, 기능 4~5개, 간단한 데이터 구조 |
| 고급 | 모듈 분리, 기능 5개 이상, 웹 UI 또는 외부 API 연동 |

---

## 평가 시 가산점 (도전 정신 15점)

- 창의적인 아이디어: 다른 팀이 생각 못 한 주제
- 기술적 도전: 40분 안에 예상보다 더 많은 것을 완성
- 프롬프트 전략: 특히 영리한 프롬프트 기법 발표 시
