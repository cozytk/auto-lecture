# CLI vs MCP 비교 분석 — 상황별 최적 도구 연동 방식 선택

## 실습 목적

CLI 방식과 MCP 방식의 구조적 차이를 비교 분석하고, 실무 시나리오별로 어떤 방식이 적합한지 판단 기준을 수립한다. 벤치마크 데이터와 실제 사례를 근거로 의사결정을 내리는 연습을 한다.

## 산출물

- CLI vs MCP 비교표 (구조, 성능, 보안, 운영 관점)
- 시나리오별 의사결정 기록서 (5개 시나리오)
- 나의 프로젝트 적용 계획서

## I DO: 시연 관찰 (약 5분)

강사가 실제 프로젝트에서 "CLI vs MCP" 의사결정을 내리는 과정을 보여줍니다.

### 시연 시나리오

> "사내 AI 챗봇에 Jira 이슈 조회 기능을 추가해야 한다. CLI로 할까, MCP 서버로 만들까?"

강사의 분석 흐름:
1. **현재 상태 파악**: Jira CLI(`jira-cli`)가 이미 설치되어 있다
2. **요구사항 분석**: AI 앱 1개(사내 챗봇)에서만 사용, 도구 2개(이슈 조회, 이슈 생성)
3. **벤치마크 데이터 참조**: CLI Token Efficiency Score 202 vs MCP 152
4. **판단**: 도구 2개 + AI 앱 1개 → CLI 래핑이 더 효율적
5. **미래 확장 고려**: 향후 Slack봇, VS Code 확장도 연결 예정이라면? → MCP로 전환 가치 있음

### 관찰 포인트

- 판단이 **이분법(CLI vs MCP)**이 아닌 **트레이드오프 분석**으로 진행되는 점
- "지금 당장"과 "6개월 후"를 모두 고려하는 점
- 벤치마크 수치를 맹신하지 않고 맥락과 함께 해석하는 점

## YOU DO: 독립 과제 (약 15분)

아래 5개 시나리오를 분석하고 의사결정 기록서를 작성하세요.

### 과제 설명

각 시나리오에 대해 아래 템플릿에 따라 분석하세요. 판단의 핵심은 **근거의 구체성**입니다.

### 시나리오 목록

1. **개인 CLI 자동화**: 개발자가 자신의 터미널에서 AI로 파일 정리를 자동화하려 한다. `ls`, `mv`, `cp` 같은 기본 명령어를 사용.

2. **팀 내부 도구 표준화**: 5명의 개발자가 각각 다른 AI 앱(Claude Desktop, Cursor, VS Code Copilot)을 쓰고 있다. 팀 공통 도구(GitHub, Slack, 내부 API) 10개를 모든 앱에서 사용하고 싶다.

3. **프로토타입 빠른 검증**: 스타트업에서 MVP를 2주 내에 만들어야 한다. AI가 외부 API 3개(결제, 이메일, SMS)를 호출해야 한다.

4. **보안이 중요한 금융 서비스**: 은행 내부 시스템에 AI 어시스턴트를 도입한다. 고객 데이터 조회, 계좌 이체 승인 등 민감한 작업을 처리한다.

5. **오픈소스 AI 도구 배포**: 커뮤니티에 공개할 AI 도구를 만든다. 다양한 AI 앱에서 설치하여 바로 사용할 수 있어야 한다.

### 작성 템플릿

```markdown
## 시나리오 N: {제목}

### 상황 분석
- 도구 개수:
- AI 앱 개수:
- 출력 정형화 필요성: (높음 / 중간 / 낮음)
- 상태 관리 필요성: (있음 / 없음)
- 보안 요구 수준: (높음 / 중간 / 낮음)
- 재사용성 요구: (높음 / 중간 / 낮음)

### 판단
- 선택: (CLI / MCP / 하이브리드)
- 핵심 근거: (1~2문장)

### 트레이드오프
- 이 선택의 장점:
- 이 선택의 단점:
- 반대 선택을 했다면 생길 문제:
```

### 제출 기준

- [ ] 5개 시나리오 모두 분석 완료
- [ ] 각 시나리오에 "핵심 근거"가 구체적 (단순히 "MCP가 좋으니까"가 아닌, 어떤 요소 때문인지 명시)
- [ ] 최소 1개 시나리오에서 "하이브리드" 선택을 고려
- [ ] 트레이드오프 항목에서 반대 선택의 문제점도 서술

### 자기 점검 기준

- 판단 근거가 가이드에서 배운 비교 축(토큰 효율성, 재사용성, 보안 등)을 활용하고 있는가?
- "정답"을 찾으려 하지 않고, 상황에 맞는 판단을 내렸는가?
- 같은 시나리오에서도 조건이 바뀌면 판단이 달라질 수 있음을 인식하고 있는가?

## 검증 방법

- 산출물이 5개 시나리오 분석을 모두 포함하는가
- 각 시나리오의 판단 근거가 구체적이고 비교 축에 기반하는가
- 트레이드오프가 양쪽 모두 서술되어 있는가
- 다른 세션(Session 1, 2)에서 배운 개념이 분석에 반영되어 있는가
