# CLI vs MCP 비교표 — 완성 예시

이 문서는 WE DO 단계에서 함께 완성한 비교표의 모범 답안입니다.

| 비교 축 | CLI 방식 | MCP 방식 | 비고 |
|---------|---------|---------|------|
| **도구 정의** | 코드에 하드코딩 | MCP 서버가 동적 제공 | MCP는 런타임에 도구 목록 변경 가능 |
| **출력 형식** | 비정형 텍스트 (stdout) | 정형 JSON | AI 파싱 정확도에 직접 영향 |
| **도구 검색** | 불가 (사전에 알아야 함) | 가능 (동적 디스커버리) | MCP 클라이언트가 서버에 도구 목록 요청 |
| **상태 관리** | 없음 (각 명령 독립) | 세션 기반 | MCP는 연결 유지 상태에서 여러 호출 |
| **보안** | 쉘 인젝션 위험 | 권한 스코프 지정 가능 | 단, MCP도 53%가 정적 API 키 의존 |
| **토큰 효율성** | TES 202 | TES 152 | CLI가 약 33% 더 효율적 (벤치마크) |
| **재사용성** | 낮음 (앱별 구현) | 높음 (표준 프로토콜) | N개 AI 앱에서 동일 MCP 서버 공유 |
| **구현 난이도** | 낮음 (subprocess 1줄) | 중간 (MCP SDK 학습 필요) | 초기 구축은 CLI가 빠름 |
