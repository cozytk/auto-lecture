# Day 1 실습 04: Architecture Decision Clinic

## 실습 목적

Session 4의 MCP / RAG / Hybrid 개념을 실제 후보 문제에 적용하여,
최종 아키텍처를 선택하고 대안을 배제한 이유까지 설명할 수 있도록 만든다.

- **연관 세션**: Session 4 - MCP·RAG·Hybrid 구조 판단
- **실습 형식**: README 중심
- **난이도**: 중급
- **예상 소요 시간**: 25분 (I DO 5분 / WE DO 8분 / YOU DO 12분)

## 트러블슈팅

| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| Hybrid를 무조건 선택함 | 만능 해법으로 오해 | 쓰기 / 검색 / 실시간성 축부터 다시 본다 |
| MCP와 Agent를 혼동함 | 용어 구분이 약함 | 여기서는 MCP를 Function Calling 중심 행동 구조로 본다 |
| 선택 이유가 추상적임 | 근거가 없음 | 요구사항 표의 Yes/No를 근거로 다시 쓴다 |
