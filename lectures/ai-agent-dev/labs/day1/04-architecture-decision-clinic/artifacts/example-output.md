# 예시 산출물: Architecture Decision Clinic

## 아키텍처 의사결정표 예시

### 후보 이름
- 코드 리뷰 자동화 Agent

### 요구사항 판단
| 항목 | Yes / No | 근거 |
|------|----------|------|
| 외부 시스템 쓰기 필요 | Yes | GitHub PR 코멘트를 작성해야 함 |
| 실시간 데이터 필요 | Yes | 현재 열려 있는 PR과 최신 커밋 기준으로 판단해야 함 |
| 대량 문서 검색 필요 | Yes | 설계 문서와 가이드라인을 찾아야 함 |
| 멀티스텝 판단 필요 | Yes | 검색 -> 비교 -> 코멘트 작성 순서 필요 |

### 최종 선택
- 선택: Hybrid
- 세부 패턴: RAG-then-Act

### 선택 이유
1. 설계 문서 검색은 RAG가 가장 적합하다.
2. PR 코멘트 작성은 외부 시스템 쓰기이므로 Agent 행동 계층이 필요하다.
3. 검색 결과에 따라 코멘트 내용이 달라지는 멀티스텝 판단이 필요하다.

### 대안 배제 이유
- 배제한 대안 1: 순수 RAG - 문서 검색만 가능하고 PR 쓰기 작업이 불가능하다.
- 배제한 대안 2: 순수 MCP - 대량 문서 검색 품질을 별도 설계해야 하므로 비효율적이다.

### Day 5 MVP 연결 메모
- MVP에서는 설계 문서 1종 + GitHub PR 코멘트 자동화까지만 구현하고, Slack 알림은 제외한다.
