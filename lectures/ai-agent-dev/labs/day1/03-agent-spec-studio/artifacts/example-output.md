# 예시 산출물: Agent Spec Studio

## Agent 기획서 예시

### 1. 목적
- 누구를 위해: 백엔드 개발팀을 위해
- 무엇을: 주간 보고서 초안 생성을 자동화한다
- 정량 목표: 180분 걸리던 작업을 10분 이내로 줄인다

### 2. 입력
| 이름 | 타입 | 필수 여부 | 예시 |
|------|------|----------|------|
| week_start | date | 필수 | 2025-03-03 |
| team_id | string | 필수 | backend-platform |

### 3. 출력
| 이름 | 타입 | 필드 / 형식 | 예시 |
|------|------|-------------|------|
| report_summary | json | summary, jira, commits, risks | weekly-report-v1 |
| report_url | string | URL | https://confluence/... |

### 4. 제약조건
| 유형 | 규칙 | 위반 시 영향 |
|------|------|-------------|
| 보안 | 개인 식별 가능한 성과 정보는 외부로 전송하지 않는다 | 개인정보 유출 |
| 비즈니스 | 미완료 티켓은 완료로 표기하지 않는다 | 보고서 신뢰 하락 |
| 기술 | 외부 API 호출은 30초 이내 타임아웃 처리한다 | 보고서 생성 지연 |

### 5. 성공 기준
| 지표 | 목표값 | 측정 방법 |
|------|-------|----------|
| 보고서 생성 시간 | 10분 이내 | 금요일 오전 평균 |
| 데이터 정확도 | 95% 이상 | 주간 수동 검토 |

### 6. 범위 경계
#### In Scope
- Jira 티켓 수집
- Git 커밋 요약
- Confluence 저장

#### Out of Scope
- 개인별 성과 평가
- 타팀 보고서 생성

## 나쁜 기획서 개선 메모 예시

| 항목 | 현재 문제 | 개선 방식 |
|------|----------|----------|
| 목적 | "개선한다" 수준으로 모호함 | 시간 절감 목표를 수치화 |
| 입력 | 필드 정의 없음 | 타입, 필수 여부, 예시 추가 |
| 출력 | 결과 형식 불명확 | JSON 구조와 URL 분리 |
| 제약조건 | 선언적 표현 | 위반 영향까지 명시 |
| 성공 기준 | 측정 불가 | 목표값과 측정 방법 추가 |
| 범위 경계 | Out of Scope 부재 | 하지 않는 일 명시 |

## 기술 설계 보완 메모 예시

| 구분 | 내용 |
|------|------|
| 권장 아키텍처 | Agent |
| 선택 이유 | Jira/Git/Confluence 쓰기와 멀티스텝 통합이 필요함 |
| 주요 에러 시나리오 1 | Jira API 타임아웃 |
| 주요 에러 시나리오 2 | Git 커밋 데이터 누락 |
| 주요 에러 시나리오 3 | Confluence 저장 실패 |
| Guardrail 아이디어 | 저장 전 필수 섹션 유무 검증 |
