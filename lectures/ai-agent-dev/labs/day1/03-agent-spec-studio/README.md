# Day 1 실습 03: Agent Spec Studio

## 실습 목적

Session 1에서 도출한 후보를 실제 구현 가능한 Agent 기획서로 변환하고,
좋은 기획서와 나쁜 기획서의 차이를 비교하여 Day 5 MVP 설계의 기반을 만든다.

- **연관 세션**: Session 3 - Agent 기획서 구조화
- **실습 형식**: README 중심
- **난이도**: 중급
- **예상 소요 시간**: 75분 (I DO 15분 / WE DO 20분 / YOU DO 40분)

## 산출물

- Agent 기획서 초안 1개
- 나쁜 기획서 개선 메모 1개
- 기술 설계 보완 메모 1개

## I DO: 시연 관찰 (약 15분)

강사가 "주간 보고서 자동화" 후보를 아래 3단계로 변환하는 과정을 시연한다.

1. 기획서 6요소 작성
2. 나쁜 기획서와 비교하며 품질 차이 설명
3. 기술 설계 보완 항목 도출

### 관찰 포인트

- 목적은 한 문장이어야 한다.
- 성공 기준은 측정 가능해야 한다.
- 제약조건은 보안/비즈니스/기술을 모두 포함해야 한다.
- 기술 설계는 기획서의 제약과 범위를 그대로 이어받아야 한다.

## 검증 방법

- 기획서 6개 구성요소가 모두 채워졌는가
- 제약조건이 최소 3종류(보안/비즈니스/기술)인가
- 성공 기준이 숫자 또는 판정 기준을 포함하는가
- 기술 설계 보완 메모가 기획서와 충돌하지 않는가

## 트러블슈팅

| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| 목적 문장이 모호함 | "개선한다" 수준에 머묾 | 시간, 정확도, 처리량 중 하나를 수치화한다 |
| Out of Scope가 비어 있음 | 범위 통제가 없음 | "하지 않는 것"을 최소 2개 적는다 |
| 제약조건이 선언적임 | 구현 연결이 안 됨 | 위반 시 영향까지 함께 적는다 |
