# 예시 산출물: Agent Opportunity Workshop

## 후보 A — 주간 보고서 자동화

- Pain: 매주 금요일 보고서 작성에 180분 소요
- 빈도: 주 1회
- 회당 시간: 180분
- 현재 방식의 문제: Jira, Git, Slack을 각각 열어 수동으로 정리해야 함

### Task
1. Jira 완료/진행중 티켓 수집
2. Git 커밋 및 PR 요약
3. Slack 주요 논의 추출
4. 보고서 템플릿에 맞게 통합

### Skill
- 외부 API 호출
- 정보 요약
- 데이터 통합
- 문서 생성

### Tool
- Jira API
- GitHub API
- Slack API
- Confluence API
- LLM API

### 적합성 판단
- 멀티스텝: Yes
- 동적 판단: Yes
- Tool 사용: Yes
- 1차 결론: Agent

## 후보 B — 사내 FAQ 챗봇

- Pain: 반복 질문 응답에 하루 90분 소요
- 빈도: 일 20회 이상
- 회당 시간: 3~5분
- 현재 방식의 문제: 문서를 사람이 찾아서 링크와 설명을 수동 제공

### Task
1. 질문 의도 파악
2. 관련 문서 검색
3. 답변 생성

### Skill
- 질의 이해
- 문서 검색
- 답변 생성

### Tool
- Vector DB
- Embedding 모델
- LLM API

### 적합성 판단
- 멀티스텝: Yes
- 동적 판단: 제한적
- Tool 사용: Yes
- 1차 결론: RAG

## 최종 후보 2개 선정 예시

| 후보 | 기대 효과 | 구현 난이도 | Agent 적합성 | 최종 선택 여부 |
|------|----------|------------|--------------|----------------|
| 주간 보고서 자동화 | 매우 큼 | 중간 | 높음 | 선택 |
| 사내 FAQ 챗봇 | 큼 | 낮음 | 중간 | 선택 |
| 단순 날짜 포맷 변환 | 작음 | 낮음 | 낮음 | 제외 |

## 패턴 매핑 예시

| 후보 | 권장 구조 | 패턴 | 선택 이유 | 배제한 대안 |
|------|----------|------|----------|------------|
| 주간 보고서 자동화 | Agent | 자동화형 | 멀티스텝 + 외부 쓰기 + 동적 판단 | RAG 배제: 검색보다 실행이 핵심 |
| 사내 FAQ 챗봇 | RAG | 분석형 | 대량 문서 검색 중심, 외부 쓰기 없음 | Agent 배제: 행동보다 검색이 핵심 |
