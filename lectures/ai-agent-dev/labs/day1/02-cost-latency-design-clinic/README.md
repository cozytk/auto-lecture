## I DO: 시연 관찰 (약 5분)

강사가 "고객 문의 분류 -> 행동 결정 -> 최종 응답" 흐름을 예시로,
각 단계에 어떤 전략을 배치해야 비용과 지연을 줄일 수 있는지 설명한다.

### 관찰 포인트

- 모든 단계에 CoT를 쓰면 비싸고 느리다.
- 모든 단계에 최고급 모델을 쓰면 과설계다.
- 분류 / 추출 / 결정 / 생성은 서로 다른 최적 전략을 가질 수 있다.

| 처리 단계 | 입력 길이 | 실패 비용 | 속도 우선 / 정확도 우선 | 모델 | 프롬프트 전략 | 추가 최적화 |
|----------|----------|----------|------------------------|------|--------------|------------|
| Step 1 | | | | | | |
| Step 2 | | | | | | |
| Step 3 | | | | | | |

아래 질문에도 답한다.

1. 가장 비용이 많이 들 단계는 어디인가?
2. 병렬화 가능한 Tool 호출은 무엇인가?
3. 요약/압축이 필요한 대화 이력 구간은 어디인가?
4. 경량 모델 실패 시 상위 모델 Fallback이 필요한 단계는 어디인가?

## 검증 방법

- 최소 3단계 이상이 구분되어 있는가
- 각 단계마다 모델과 전략 선택 이유가 있는가
- 비용 최적화와 정확도 보호가 동시에 고려되었는가

## 트러블슈팅

| 증상 | 원인 | 해결 방법 |
|------|------|----------|
| 모든 단계가 같은 모델 | 단계별 목표가 안 나뉨 | 분류 / 판단 / 생성으로 다시 나눈다 |
| CoT를 모든 곳에 사용 | 복잡도 판단 부재 | 단순 분류 단계는 Zero-shot/Few-shot으로 줄인다 |
| 비용 절감만 있고 안정성 없음 | Fallback 미설계 | 고위험 단계에 상위 모델 재요청 조건을 추가한다 |
