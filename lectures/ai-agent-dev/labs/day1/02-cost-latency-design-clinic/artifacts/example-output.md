# 예시 산출물: Cost-Latency Design Clinic

## 요청 단계별 설계 예시 — 고객 문의 처리 Agent

| 처리 단계 | 입력 길이 | 실패 비용 | 속도 우선 / 정확도 우선 | 모델 | 프롬프트 전략 | 추가 최적화 |
|----------|----------|----------|------------------------|------|--------------|------------|
| Step 1 의도 분류 | 짧음 | 낮음 | 속도 우선 | gpt-4.1-mini | Zero-shot | 캐싱 |
| Step 2 행동 결정 | 중간 | 높음 | 정확도 우선 | gpt-4o | Structured Output | 검증 규칙 |
| Step 3 예외 판단 | 김 | 매우 높음 | 정확도 우선 | claude-sonnet | CoT | Fallback |

## 분석 메모 예시

1. 가장 비용이 많이 드는 단계는 Step 3이다. 긴 맥락 + CoT + 상위 모델 조합이기 때문이다.
2. 병렬화 가능한 Tool 호출은 고객 주문 조회와 배송 상태 조회다.
3. 5턴 이상 대화가 이어질 경우 Step 2 이전에 요약/압축이 필요하다.
4. Step 1 confidence가 낮으면 Step 2로 넘기기 전에 상위 모델 재분류를 수행한다.
