# 모범 답안: 구조 결정 (과제 3)

> 이 파일은 YOU DO 과제 3의 모범 답안이다.
> 고객 문의 자동 분류 Agent의 구조 결정 과정을 설명한다.

---

## 구조 결정: Hybrid (RAG + MCP)

### 최종 선택: Hybrid

---

### 선택 이유

**이유 1: 외부 시스템 조작이 필수적이다**

이메일 수신(IMAP)과 발송(SMTP)은 외부 API 호출이 필요하다.
RAG만으로는 이메일을 실제로 읽거나 발송할 수 없다.
MCP Tool이 반드시 필요한 이유다.

**이유 2: 유사 케이스 검색은 RAG가 최적이다**

과거 문의 이력(수천 건)을 의미 기반으로 검색해야 한다.
단순 키워드 검색으로는 "포장이 찢어졌어요"와 "배송 박스가 손상됐어요"가 유사 케이스임을 알 수 없다.
임베딩 기반 유사도 검색(RAG)이 이 문제에 가장 적합하다.

**이유 3: 두 요소가 분리될 수 없다**

RAG 결과(유사 케이스)가 LLM의 답변 초안 생성(MCP Tool 호출 전)에 필요하다.
두 구조가 순차적으로 연결되어야 하므로 Hybrid가 불가피하다.

---

### 비선택 구조의 단점

**MCP만 사용할 경우**:
- 과거 유사 케이스를 검색하려면 전통적인 키워드 검색 Tool을 사용해야 한다
- 의미 기반 검색이 불가능해 관련성 낮은 케이스가 검색될 수 있다
- 결과적으로 답변 초안 품질이 저하된다

**RAG만 사용할 경우**:
- 이메일 수신·발송이 불가능하다 (외부 API 호출 필요)
- 실시간 처리가 어렵다 (RAG는 색인된 데이터만 검색)
- 이메일 라우팅 등 실제 액션을 취할 수 없다

---

### 예상 비용 구조

| 비용 항목 | 발생 조건 | 추정 규모 |
|-----------|-----------|-----------|
| LLM 호출 비용 | 이메일 분류 + 초안 생성 (건당 2회) | 월 10만 건 시 주요 비용 |
| 임베딩 비용 | 신규 케이스 추가 시 재임베딩 | 초기 구축 후 증분만 발생 |
| 벡터 DB 비용 | 저장 + 검색 쿼리 | 데이터 규모에 따라 선형 증가 |
| API 호출 비용 | IMAP/SMTP는 대부분 무료 또는 저렴 | 미미한 수준 |

**비용 최적화 전략**:
- 분류 단계: 작은 모델 사용 (단순 분류 태스크)
- 초안 생성: 중간 모델 사용 (품질 중요)
- 임베딩: 배치 처리로 단가 절감
- 캐싱: 동일 또는 매우 유사한 문의는 캐시 응답 활용

---

### 구조 다이어그램

```
[이메일 수신 - MCP Tool]
        ↓
[LLM 분류 - MCP Tool Use]
        ↓
[유사 케이스 검색 - RAG]
        ↓
[답변 초안 생성 - MCP Tool Use (RAG 결과 포함)]
        ↓
[이메일 발송 - MCP Tool]
```

RAG와 MCP가 순차적으로 결합되는 구조다.
RAG 검색 결과가 MCP Tool Use의 Context로 사용된다.

### 확장성 고려

**단기 (0-6개월)**:
- Hybrid 구조 MVP 구현
- 분류 정확도 모니터링

**중기 (6-12개월)**:
- 자동 답변 비율 확대 (현재 70% → 90%)
- 언어별 분류 모델 추가

**장기 (12개월+)**:
- 다국어 지원
- 실시간 학습 (새 케이스 자동 색인)
- Planner형 Agent로 복잡한 문의 처리 확장
